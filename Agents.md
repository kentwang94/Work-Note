## 维护规则

### 1. 历史文档归档
- 所有 2018 年及之前编辑的文档已归档到 `2018/` 文件夹中
- 归档文件夹名称为最后编辑的年份

### 2. 新增内容规范
- **每日记录格式**: `yyyy-mm-dd.md`（例如：`2024-01-15.md`）
- **月度组织**: 每个月创建一个文件夹，格式为 `yyyy-mm`（例如：`2024-01/`）
- 每日新增的内容放在对应月份的文件夹中

### 3. 文件命名规范
- 日期格式：使用 ISO 8601 标准的日期格式 `YYYY-MM-DD`
- 文件名只包含小写字母、数字和连字符
- 文件扩展名必须为 `.md`

### 4. 文档内容建议
- 每篇文档应包含清晰的标题
- 建议添加关键词（keywords）便于检索
- 代码片段请使用 Markdown 代码块语法
- 引用外部资源时请提供链接

### 5. GitHub Pages 配置
- 本项目支持通过 GitHub Pages 访问
- 确保在 GitHub 仓库设置中启用 Pages 功能
- 默认使用 `main` 分支作为发布源

### 6. 提交流程
1. 在对应的月份文件夹中创建新的日期文件
2. 编写技术总结内容
3. 提交更改并推送至仓库
4. GitHub Pages 会自动更新
